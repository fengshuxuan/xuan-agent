# 02. MVP 范围定义

## MVP 目标

第一版只做一个稳定闭环：

```text
用户聊天 / 上传文件
  → Agent 理解任务
  → 调用文件工具或代码工具
  → 在 workspace / sandbox 中执行
  → 返回结果和生成文件
```

## MVP 用户故事

### 用户故事 1：普通聊天

作为用户，我可以向 Agent 提问，让它直接回答，不一定调用工具。

验收标准：

- 用户输入文本后能收到回复
- 会话能保持基本上下文
- 不需要工具时不会乱调用工具

### 用户故事 2：上传并查看文件

作为用户，我可以上传文件，并让 Agent 查看当前有哪些文件。

验收标准：

- 支持上传文件到 session workspace
- 支持列出当前 workspace 文件
- Agent 只能看到当前 session 文件

### 用户故事 3：读取文件

作为用户，我可以让 Agent 读取一个文本文件并总结。

验收标准：

- 支持 txt / md / csv / json / py 等文本文件
- 文件路径必须限制在 workspace 内
- 文件不存在时返回明确错误

### 用户故事 4：执行 Python

作为用户，我可以让 Agent 执行一段 Python 代码。

验收标准：

- 代码在 Docker 沙箱中运行
- 返回 stdout / stderr / exit_code
- 限制执行时间
- 限制访问目录

### 用户故事 5：生成结果文件

作为用户，我可以让 Agent 生成一个新文件。

验收标准：

- 支持写入 workspace 输出文件
- 返回生成文件路径
- 支持前端展示下载链接

## MVP 功能清单

| 模块 | 功能 | 优先级 |
|---|---|---|
| Chat | 发送消息 | P0 |
| Chat | 返回 Agent 回复 | P0 |
| File | 上传文件 | P0 |
| File | 列出文件 | P0 |
| File | 读取文本文件 | P0 |
| File | 写入文本文件 | P0 |
| Agent | 工具选择 | P0 |
| Agent | 工具执行循环 | P0 |
| Sandbox | 执行 Python | P0 |
| Sandbox | 超时限制 | P0 |
| Log | 记录工具调用 | P1 |
| Data | CSV / Excel 分析 | P1 |
| Chart | 生成图表 | P1 |

## 暂不纳入 MVP

- 多 Agent 分工
- MCP 插件市场
- 自动浏览器操作
- 邮件发送
- 长期记忆
- 团队权限
- 计费系统
- 复杂任务队列

## MVP 成功标准

只要能稳定完成下面三个任务，就算第一版可用：

1. 上传 CSV，让 Agent 读取并总结字段
2. 让 Agent 执行 Python 代码并返回结果
3. 让 Agent 生成一个新文件并返回路径
