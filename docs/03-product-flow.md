# 03. 产品流程设计

## 核心交互流程

```text
用户进入聊天页
  ↓
输入问题或上传文件
  ↓
后端创建 / 读取 session
  ↓
Agent 判断是否需要工具
  ↓
工具执行：文件 / Python / 后续 MCP
  ↓
Agent 整理结果
  ↓
前端展示回复、工具状态和下载文件
```

## 页面结构

MVP 阶段只需要一个主页面：

```text
┌────────────────────────────┐
│ Xuan Agent                 │
├────────────────────────────┤
│ 对话消息区                 │
│ - 用户消息                 │
│ - Agent 回复               │
│ - 工具执行状态             │
├────────────────────────────┤
│ 文件列表 / 生成文件         │
├────────────────────────────┤
│ 输入框 + 上传按钮 + 发送按钮 │
└────────────────────────────┘
```

## 消息类型

| 类型 | 说明 |
|---|---|
| user_message | 用户输入 |
| assistant_message | Agent 回复 |
| tool_call | 工具调用记录 |
| tool_result | 工具返回结果摘要 |
| file_result | 生成文件 |
| error | 错误提示 |

## 典型流程：分析文件

1. 用户上传 `sales.csv`
2. 前端调用 `/api/files/upload`
3. 后端保存到 `workspace/{session_id}/uploads`
4. 用户输入：`帮我分析这个文件`
5. 前端调用 `/api/chat`
6. Agent 调用 `list_files`
7. Agent 调用 `read_text_file` 或 `execute_python`
8. Agent 返回分析结果
9. 如果生成文件，前端展示下载链接

## 典型流程：执行代码

1. 用户输入 Python 代码执行请求
2. Agent 判断需要调用 `execute_python`
3. 后端将代码写入临时脚本
4. Docker 沙箱执行脚本
5. 沙箱返回 stdout / stderr / exit_code
6. Agent 总结结果

## 错误提示原则

错误信息应做到：

- 告诉用户发生了什么
- 告诉用户哪些文件 / 代码出错
- 不暴露服务器敏感路径
- 给出下一步建议

示例：

```text
执行失败：代码运行超过 10 秒，已自动终止。
你可以减少数据量，或让我帮你优化代码后重试。
```

## 前端状态展示

Agent 调用工具时，前端可以展示：

```text
正在查看文件...
正在执行 Python...
正在生成结果文件...
完成。
```

MVP 阶段可以先通过普通消息模拟，后续改成流式事件。
