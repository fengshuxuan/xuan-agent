# 01. 需求分析

## 项目定位

Xuan Agent 是一个通用型个人工作 Agent，第一阶段聚焦于：

- 自然语言聊天
- 文件上传、读取、写入和生成
- Python 代码沙箱执行
- 基于工具调用完成多步骤任务

它不是单纯聊天机器人，而是一个带工作区、工具系统和执行能力的任务助手。

## 目标用户

| 用户类型 | 典型需求 |
|---|---|
| 办公用户 | 整理文件、生成表格、总结文档 |
| 数据分析用户 | 分析 Excel / CSV、画图、生成报告 |
| 开发者 | 执行代码、调试脚本、批量处理文件 |
| 内容创作者 | 整理素材、生成文案、批量处理内容 |

## 核心场景

### 场景 1：Excel 数据分析

用户上传 Excel，并输入：

> 帮我分析这个 Excel，找出销售额最高的产品，并画一张图。

Agent 应执行：

1. 识别用户意图
2. 查看当前 workspace 文件
3. 读取 Excel
4. 用 Python 分析数据
5. 生成图表文件
6. 返回结论和下载链接

### 场景 2：代码执行

用户输入：

> 帮我运行这段 Python，并告诉我哪里报错。

Agent 应执行：

1. 识别代码执行任务
2. 将代码放入沙箱
3. 执行并收集 stdout / stderr
4. 总结错误原因
5. 必要时给出修复版代码

### 场景 3：文件整理

用户输入：

> 帮我把当前文件夹里的 csv 合并成一个 Excel。

Agent 应执行：

1. 列出 workspace 文件
2. 找到 CSV 文件
3. 执行 Python 合并
4. 生成 Excel
5. 返回下载路径

## 功能需求

### P0 必须完成

- 聊天接口
- 文件上传
- workspace 管理
- 文件列表查看
- 文本文件读取
- 文本文件写入
- Python 沙箱执行
- 执行结果返回
- 基础错误处理

### P1 应尽快完成

- Excel / CSV 文件分析
- 图表生成
- 任务状态记录
- 工具调用日志展示
- 生成文件下载

### P2 后续扩展

- 长期记忆
- RAG 文档检索
- MCP 工具接入
- 多 Agent 协作
- 浏览器自动化
- 邮件 / 日历集成

## 非功能需求

- 安全：代码执行必须隔离
- 可观测：记录每次工具调用
- 可恢复：工具失败后可以重试
- 可维护：工具层和模型层解耦
- 可扩展：后续能接 MCP 和多 Agent

## 边界与限制

第一版不做：

- 自动删除用户文件
- 自动发送外部消息
- 自动操作浏览器
- 多用户权限系统
- 复杂任务队列
- 生产级支付 / 账号体系

第一版只保证当前 session 内的文件和代码处理闭环。
